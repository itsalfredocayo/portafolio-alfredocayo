# Proyecto de Portafolio — Alfredo Cayo

## Herramientas instaladas
- **Cursor IDE** — Editor de código con IA integrada (cursor.com)
- **Claude Code for VS Code** — Extensión oficial de Anthropic para asistencia con código
- **Codex** — Extensión de OpenAI para generación de código
- **Git for Windows** — Sistema de control de versiones (git-scm.com)

## Pasos completados
1. Instalé Cursor IDE desde cursor.com
2. Instalé la extensión Claude Code for VS Code desde el Marketplace de Cursor
3. Instalé la extensión Codex en Cursor
4. Creé una cuenta en GitHub y un repositorio público llamado portafolio-alfredocayo
5. Instalé Git for Windows al descubrirque no estaba disponible en mi sistema
6. Cloné el repositorio usando git clone desde la terminal de Cursor
7. Edité este README.md documentando todo el proceso
8. Hice commit y push de los cambios a GitHub

## Problemas encontrados y cómo los resolví

### 1. No sabía cómo iniciar sesión en Claude Code
Al instalar la extensión no era obvio cómo autenticarme. Investigué y descubrí
que había que abrir el panel de Claude Code desde la barra lateral y buscar
el botón de login.

### 2. Claude Code requiere suscripción de pago
Al intentar iniciar sesión con mi cuenta gratuita de claude.ai, el sistema
me mostró el mensaje "Se requiere Claude Max o Pro para conectarse a Claude Code".
Lo resolví creando una cuenta en console.anthropic.com y generando una API Key
gratuita para autenticarme por esa vía.

### 3. Git no estaba instalado en mi computadora
Al intentar clonar el repositorio desde Cursor, apareció el error:
"git isn't available in your PowerShell session (git: no se reconoce)".
Lo resolví descargando e instalando Git desde git-scm.com y reiniciando
Cursor para que reconociera el nuevo programa.

### 4. Confusión entre la terminal de Cursor y el chat de IA
Intenté escribir comandos de git en el chat del asistente de IA en lugar
de en la terminal real. Lo resolví identificando que la terminal se abre
con View → Terminal en la barra superior.

### 5. No encontraba la carpeta clonada
Después de clonar el repositorio no sabía dónde se había guardado la carpeta.
Lo resolví usando el comando pwd en la terminal para ubicar la ruta exacta
(D:\Cursor\portafolio-alfredocayo) y luego abrirla desde Cursor.

## Reflexión
Este proceso me enseñó que los errores e imprevistos son parte normal del
trabajo con herramientas de desarrollo. Cada problema encontrado fue una
oportunidad para buscar soluciones de forma independiente, que es una
habilidad clave en el desarrollo de software.
